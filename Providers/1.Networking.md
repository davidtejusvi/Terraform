# 1️⃣ Amazon VPC

## What is VPC?

A Virtual Private Cloud (VPC) is your own isolated network inside AWS where you can launch AWS resources securely.

Think of it as your company's private data center in the cloud.

### Real-Time Example

An e-commerce company creates one VPC for its production environment.

Inside the VPC:

- Public Subnets host the Load Balancer.
- Private App Subnets host EC2 instances.
- Private DB Subnets host Amazon RDS.

This prevents direct internet access to application servers and databases.

### Terraform

```hcl
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_support   = true
  enable_dns_hostnames = true

  tags = {
    Name = "Production-VPC"
  }
}
```

### Best Practices

- One VPC per environment
- Enable DNS Hostnames
- Plan CIDR ranges before deployment
- Avoid overlapping CIDR blocks

# 2️⃣ Internet Gateway

## What is an Internet Gateway?

An Internet Gateway connects your VPC to the public internet.

Without an Internet Gateway, users cannot access your website.

### Real-Time Example

Users open:

```
https://shop.example.com
```

Traffic flows like this:

```
User
   │
Internet
   │
Internet Gateway
   │
Application Load Balancer
```

### Terraform

```hcl
resource "aws_internet_gateway" "igw" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "Production-IGW"
  }
}
```

# 3️⃣ Public Subnet

## What is a Public Subnet?

A subnet with a route to the Internet Gateway.

Resources inside it can receive traffic from the internet.

### Real-Time Example

Typical resources:

- Application Load Balancer
- NAT Gateway
- Bastion Host

Never place databases inside a public subnet.

### Terraform

```hcl
resource "aws_subnet" "public_a" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.1.0/24"
  availability_zone       = "ap-south-1a"
  map_public_ip_on_launch = true

  tags = {
    Name = "Public-Subnet-A"
  }
}
```

# 4️⃣ Private Application Subnet

## Purpose

Stores application servers.

Examples:

- EC2
- ECS
- EKS Worker Nodes

These servers should never have public IP addresses.

### Real-Time Example

A banking application hosts Node.js APIs in private subnets.

Only the Load Balancer can communicate with them.

### Terraform

```hcl
resource "aws_subnet" "private_app_a" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.11.0/24"
  availability_zone = "ap-south-1a"

  tags = {
    Name = "Private-App-A"
  }
}
```

---

# 5️⃣ Private Database Subnet

## Purpose

Stores databases securely.

Examples:

- Amazon RDS
- Amazon Aurora

These subnets should never have internet access.

### Real-Time Example

Customer payment data is stored in Amazon RDS.

Only application servers can connect to the database.

### Terraform

```hcl
resource "aws_subnet" "private_db_a" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.21.0/24"
  availability_zone = "ap-south-1a"

  tags = {
    Name = "Private-DB-A"
  }
}
```

# 6️⃣ NAT Gateway

## What is a NAT Gateway?

Allows private resources to access the internet without exposing them to inbound internet traffic.

### Real-Time Example

Private EC2 instances:

- Install OS updates
- Download packages
- Access AWS APIs

without having public IP addresses.

### Terraform

```hcl
resource "aws_nat_gateway" "nat" {
  allocation_id = aws_eip.nat.id
  subnet_id     = aws_subnet.public_a.id

  tags = {
    Name = "Production-NAT"
  }
}
```

# 7️⃣ Route Tables

## Purpose

Controls where network traffic goes.

### Public Route Table

```
Destination      Target
0.0.0.0/0        Internet Gateway
```

### Private Route Table

```
Destination      Target
0.0.0.0/0        NAT Gateway
```

### Terraform

```hcl
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.igw.id
  }

  tags = {
    Name = "Public-RT"
  }
}
```

# ✅ Interview Questions

### Why do we use private subnets?

To protect application servers and databases from direct internet access.

---

### Why do we need a NAT Gateway?

To allow private resources to access the internet for updates and external services without exposing them to inbound internet traffic.

---

### Can Amazon RDS be deployed in a public subnet?

It can, but it is strongly discouraged for production workloads. Databases should typically be placed in private subnets and accessed only by application servers.

---

### Why use multiple Availability Zones?

To improve high availability and fault tolerance. If one Availability Zone becomes unavailable, resources in the other zone can continue serving traffic.
